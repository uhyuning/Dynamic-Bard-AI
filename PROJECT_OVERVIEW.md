# Dynamic Bard AI — 프로젝트 전체 파일 설명서

> 이 문서는 개발을 처음 접하는 분도 각 파일이 무슨 역할을 하는지 이해할 수 있도록 작성되었습니다.
> 파일을 수정할 때 "이게 뭐하는 파일이지?" 싶을 때 여기서 찾아보세요.

---

## 프로젝트가 하는 일 (한 줄 요약)

> 사용자가 AI 캐릭터와 대화하면, AI가 대화 내용으로 기분을 파악해서 어울리는 음악을 틀어주고,
> 맵/분위기도 함께 바꿔주는 언리얼5 게임입니다.
> AI는 사용자가 노래를 스킵하거나 좋아요를 누를 때마다 더 잘 맞는 곡을 추천하도록 학습합니다.

### AIServer/ 폴더 — AI 서버 (프로젝트의 심장)

> Python으로 만든 웹 서버입니다.
> 언리얼5 게임과 GPT, 음악 추천 AI를 연결해주는 중간 다리 역할을 합니다.

---

#### `AIServer/server.py` ★ 가장 중요한 파일

**역할**: 모든 것을 연결하는 메인 서버

언리얼5에서 사용자가 채팅을 보내면 이 파일이 받아서 처리합니다.

**이 파일이 하는 일 순서:**
1. 사용자 메시지 수신
2. GPT에게 "이 사람 기분이 어때?" 물어봄
3. RL AI에게 "어떤 음악 장르가 좋을까?" 결정하게 함
4. Spotify에서 그 장르의 실제 곡 검색
5. 언리얼5로 결과 전송 (AI 대사 + 음악 정보)

**서버 주소**: `http://localhost:8000`

**주요 API 목록:**
| 주소 | 역할 |
|------|------|
| `GET  /` | 서버가 켜져 있는지 확인 |
| `POST /chat` | 채팅 메시지 처리 (핵심) |
| `POST /feedback` | 노래 피드백 수신 (좋아요/스킵) |
| `GET  /status` | AI 학습 현황 확인 |
| `GET  /now-playing` | 현재 재생 중인 곡 확인 |
| `POST /act` | 언리얼5 위치 정보로 행동 결정 |

---

#### `AIServer/gpt_client.py`

**역할**: ChatGPT와 대화하는 담당

사용자 메시지를 받아서 GPT API에 보내고, 두 가지를 동시에 받아옵니다:
1. AI가 사용자에게 할 **대화 응답** (한국어)
2. 사용자의 **감정 점수** (-1.0 ~ 1.0, 부정~긍정)
3. 사용자의 **에너지 수준** (0.0 ~ 1.0, 차분~활발)

> **예시**: 사용자가 "오늘 진짜 힘들었어"라고 하면
> → 응답: "많이 힘들었겠다, 오늘 고생했어"
> → 감정: -0.7 (꽤 부정적)
> → 에너지: 0.2 (많이 지쳐있음)

---

#### `AIServer/spotify_client.py`

**역할**: Spotify에서 실제 음악을 찾아오는 담당

RL AI가 "잔잔한 음악 틀어줘"라고 결정하면, 이 파일이 Spotify에서
실제로 그 분위기에 맞는 곡을 검색해서 가져옵니다.

**Spotify API 연동 상태에 따른 동작:**
- **Spotify 연결됨**: 실제 Spotify에서 곡 검색 → 30초 미리듣기 URL 반환
- **Spotify 미연결**: Mock 모드로 자동 전환 → 가상의 곡 목록 반환 (서버는 계속 작동)

**AI 행동 → 음악 스타일 변환표:**
| AI 행동 | 검색 스타일 | 예시 |
|--------|-----------|------|
| 0: 잔잔 | 저에너지, 어쿠스틱 | Lo-fi, Ambient |
| 1: 신남 | 고에너지, 밝음 | K-Pop, Pop |
| 2: 감성 | 중간, 서정적 | Indie, R&B |
| 3: 집중 | 잔잔, 기악 위주 | Classical, Piano |

#### `AIServer/spotify_mock.py`

**역할**: Spotify가 없을 때 사용하는 가짜 음악 데이터 생성기

현재는 `spotify_client.py` 내부에 Mock 기능이 통합되어 있으므로
이 파일은 초기 버전 참고용으로 남아있습니다.

#### `AIServer/inference.py`

**역할**: 모델 추론 스크립트 (현재 미사용, 자리만 잡아둔 파일)

나중에 별도 추론 로직이 필요할 때 여기에 추가할 예정입니다.
지금은 건드리지 않아도 됩니다.

### RL_Trainer/ 폴더 — 강화학습 (AI가 공부하는 곳)

> "강화학습(Reinforcement Learning)"이란 AI가 시행착오를 통해 스스로 학습하는 방법입니다.
> 사람이 게임을 하면서 "이건 좋은 선택이었네", "이건 실수였네"를 깨닫는 것과 같습니다.
> 이 프로젝트에서는 사용자가 노래를 좋아하면 보상(+), 스킵하면 패널티(-)를 줍니다.

---

#### `RL_Trainer/train.py` ★ 강화학습 핵심 파일

**역할**: AI가 어떤 음악을 추천할지 학습하는 핵심 엔진

두 가지 방식으로 작동합니다:

**방식 1 - 오프라인 학습** (단독 실행할 때):
```
python RL_Trainer/train.py
```
시뮬레이션 데이터로 미리 연습합니다. 실제 사용자 없이도 기초 학습 가능.

**방식 2 - 온라인 학습** (서버가 실행 중일 때):
실제 사용자와 대화하면서 실시간으로 학습합니다.
서버에서 자동으로 호출되므로 별도 실행 불필요.

**AI가 학습에 사용하는 정보 (상태 벡터, 12가지):**
| 번호 | 정보 | 설명 |
|-----|------|------|
| 0 | 사용자 감정 | -1(매우 슬픔) ~ +1(매우 기쁨) |
| 1 | 사용자 에너지 | 0(매우 지침) ~ 1(매우 활발) |
| 2 | 음악 BPM | 현재 곡의 빠르기 |
| 3 | 음악 에너지 | 현재 곡이 얼마나 신나는지 |
| 4 | 음악 긍정도 | 현재 곡이 얼마나 밝은지 (Spotify valence) |
| 5~8 | 음악 장르 | 잔잔/신남/감성/집중 중 무엇인지 |
| 9 | 대화 횟수 | 지금까지 몇 번 대화했는지 |
| 10 | 세션 시간 | 얼마나 오래 대화했는지 |
| 11 | 이전 행동 | 직전에 어떤 음악을 추천했는지 |

**보상 시스템:**
| 사용자 행동 | AI가 받는 점수 | 의미 |
|-----------|-------------|------|
| 좋아요 클릭 | +2.0 | "잘 맞췄어!" |
| 다시 듣기 | +1.5 | "정말 좋은 선택이었어!" |
| 완청 (끝까지 들음) | +0.7 | "나쁘지 않았어" |
| 스킵 | -1.5 | "이건 아니야!" |

---

#### `RL_Trainer/models/model.py`

**역할**: AI의 두뇌 구조 정의 (신경망)

`MasterAgent`라는 이름의 딥러닝 모델입니다.
12가지 상황 정보를 입력받아, 4가지 음악 방향 중 하나를 선택합니다.

```
입력 (12개 숫자)
    ↓
[두뇌 레이어 256개 뉴런]
    ↓
[두뇌 레이어 256개 뉴런]
    ↓
출력: 음악 방향 (4가지 점수)  +  세부 옵션 (10가지 점수)
```

이 파일은 모델의 "뼈대"만 정의합니다. 실제 학습은 `train.py`가 담당합니다.

---

#### `RL_Trainer/memory.py`

**역할**: AI의 경험 저장소 (우선순위 기억)

AI가 겪은 경험들을 저장했다가 나중에 꺼내 공부합니다.
단순히 저장하는 게 아니라, **더 중요한 경험을 더 자주 복습**합니다.

> 비유: 시험 공부할 때 틀린 문제를 더 자주 보는 것과 같습니다.
> AI가 틀린 선택(큰 오차)일수록 그 기억을 더 자주 학습에 사용합니다.

최대 10,000개의 경험을 저장할 수 있습니다.

#### `Shared/protocol.py`

**역할**: 서버와 언리얼5 사이에서 주고받는 데이터 형식 정의

서버와 게임이 서로 다른 언어로 소통하면 문제가 생기므로,
"우리 이 형식으로 데이터를 주고받자"고 약속해둔 파일입니다.

**정의된 데이터 형식:**
- `ChatMessage`: 사용자 → 서버로 보내는 채팅 메시지
- `ChatResponse`: 서버 → 게임으로 보내는 AI 응답 (대사 + 음악 정보)
- `SpotifyTrack`: 추천된 Spotify 트랙 정보 (곡명, 아티스트, 재생 URL 등)
- `FeedbackMessage`: 게임 → 서버로 보내는 노래 피드백 (좋아요/스킵)
- `EnvState`: 게임 → 서버로 보내는 캐릭터 위치/속도 정보
- `AIResponse`: 서버 → 게임으로 보내는 행동 결정

##### `BardAI/BardAIManager.h` / `BardAIManager.cpp`

**역할**: 언리얼5에서 AI 서버와 통신하는 컴포넌트

게임 캐릭터에 "붙이면" 자동으로 서버와 통신합니다.

**UE5 에디터에서 하는 일:**
1. 원하는 캐릭터 Blueprint 열기
2. Components 패널에서 `BardAIManager` 추가
3. 아래 이벤트들을 원하는 기능에 연결

**제공하는 이벤트 (Blueprint에서 연결 가능):**
| 이벤트 | 언제 발생 | 연결할 것 |
|--------|---------|---------|
| `OnChatReply` | AI 응답 도착 | 말풍선, 자막 표시 |
| `OnMusicChange` | 음악 장르 변경 | 사운드 에셋 교체 |
| `OnAtmosphereChange` | 감정/에너지 변화 | 조명, 안개, 날씨 변경 |
| `OnRequestFailed` | 서버 오류 | 에러 메시지 표시 |

**제공하는 함수 (Blueprint에서 호출 가능):**
- `SendChatMessage(메시지)` — 채팅 전송
- `SendFeedback(트랙ID, 행동번호, "liked"/"skipped", 청취비율)` — 피드백 전송
- `PingServer()` — 서버 연결 확인

#### `master_agent_brain.pth`

**역할**: AI의 학습된 기억 저장 파일

AI가 학습한 내용이 이 파일에 저장됩니다.
이 파일이 있으면 서버를 껐다 켜도 이전 학습 내용이 유지됩니다.

> 비유: 시험 공부를 했을 때 그 내용이 저장된 노트 같은 것입니다.
> 이 파일을 삭제하면 AI가 처음부터 다시 배워야 합니다.

자동으로 생성되고 업데이트됩니다. 직접 건드리지 않아도 됩니다.

**역할**: 프로젝트 소개 문서 (기존 작성본)

## 서버 실행 방법

```bash
# 1. 라이브러리 설치 (최초 1회)
pip install -r requirements.txt

# 2. AI 서버 시작
python AIServer/server.py

# 3. (선택) 오프라인 사전 학습 실행
python RL_Trainer/train.py
```

서버가 켜지면 브라우저에서 `http://localhost:8000` 을 열어서
`"status": "online"` 이 보이면 성공입니다.

## 자주 묻는 질문

**Q. Spotify 연결 없이도 되나요?**
A. 됩니다. `.env`에 Spotify 키를 입력하지 않으면 자동으로 Mock 모드로 전환됩니다.
   Mock 모드에서는 가상의 곡 정보로 동작하며, 서버 기능은 100% 동일합니다.

**Q. `.pth` 파일이 뭔가요?**
A. AI의 학습된 두뇌 파일입니다. PyTorch라는 딥러닝 라이브러리가 사용하는 형식입니다.
   자동으로 저장되므로 신경 쓰지 않아도 됩니다.

**Q. 파일을 잘못 수정했어요.**
A. Git을 사용 중이라면 `git status`로 변경된 파일을 확인하고
   `git checkout -- 파일명`으로 원래 상태로 되돌릴 수 있습니다.

**Q. 언리얼5 작업은 어디서 시작하나요?**
A. `Dynamic_Bard_AI_UE5/ARTIST_GUIDE.md` 파일을 먼저 읽어보세요.
   에셋 교체 방법이 단계별로 정리되어 있습니다.
